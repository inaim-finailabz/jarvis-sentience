# NewAIConcept

بناء نظام ذكاء اصطناعي متعدد الوسائط من الصفر لفهم كيفية عمل نماذج اللغة الكبيرة الحديثة — الانتباه، الرؤية، الاستدلال، العملاء، وشبكات العملاء المتعددة — باستخدام MLX على Apple Silicon.

كل مرحلة مستقلة بذاتها: كود قابل للتشغيل + درس تفاعلي يُظهر ما يفعله النموذج داخليًا.

---

## المتطلبات الأساسية

- Mac مع Apple Silicon (M1/M2/M3/M4)
- Python 3.14 عبر Homebrew (`/opt/homebrew/bin/python3.14`)
- MLX 0.31+ مثبت مسبقًا
- اختياري: `ANTHROPIC_API_KEY` لعروض توضيحية مباشرة في المرحلتين 4/5

## الفكرة الجوهرية (اقرأها أولًا)

كل شيء في نموذج لغوي كبير حديث — الاستدلال، الرؤية، الشخصيات، استخدام الأدوات — يُختزل في عملية واحدة:

> **التنبؤ بالرمز التالي، مشروطًا بجميع الرموز السابقة.**

الرمز مجرد رقم يمثل جزءًا من البيانات: حرف، جزء من كلمة، قطعة صورة، رمز صوتي، بقية بروتين. المحوّل لا يهتم بما يمثله الرمز. يتعلم أنماطًا إحصائية بين الرموز من بيانات التدريب.

هذا يعني:
- **بيانات التدريب هي هندسة الميزات بالفعل.** أنت تحدد ما يعرفه النموذج بما تضعه في بيانات التدريب.
- **التوجيهات (Prompts) ليست تعليمات.** إنها رموز تُنشّط أنماطًا تعلمها النموذج مسبقًا.
- **لا يوجد وحدة استدلال، وحدة رؤية، أو وحدة ذاكرة.** هناك آلية واحدة فقط — الانتباه — تُطبَّق على تسلسل رموز من أي مصدر.
- **الحجم هو الفارق الوحيد** بين MiniGPT الخاص بك وClaude. نفس البنية. عدد مختلف من المعاملات وحجم بيانات التدريب.

## المرحلة 1 — جوهر المحوّل

### السؤال
كيف يعمل نموذج اللغة فعليًا على المستوى الرياضي؟

### الإجابة
كل رمز ينتبه لكل رمز سابق، يحسب مزيجًا موزونًا من قيمها، يحدّث تمثيله الخاص، ويكرر ذلك عبر N طبقات. في النهاية، طبقة خطية تعكس تمثيل كل رمز على توزيع احتمالي فوق المفردات. التدريب يضبط الأوزان بحيث يحصل الرمز التالي الصحيح على احتمالية أعلى. هذا هو الآلية بالكامل.

### ما تبنيه
- `scaled_dot_product_attention` — العملية الجوهرية لكل نموذج LLM تم بناؤه على الإطلاق
- `MultiHeadAttention` — H من أنماط الانتباه المتوازية، كل منها في فضاء جزئي بعده d_model/H
- `TransformerBlock` — انتباه + MLP للتغذية الأمامية + اتصالات متبقية + LayerNorm
- `MiniGPT` — نموذج لغوي كامل على مستوى الأحرف مدرَّب على أعمال شكسبير (~840k معامل)

### المفاهيم الرئيسية

| المفهوم | ما هو |
|---|---|
| التضمين (Embedding) | كل حرف → متجه من 128 رقمًا. الأحرف المتشابهة → متجهات متشابهة بعد التدريب |
| الانتباه (Attention) | الرمز i يُقيّم كل رمز 0→i بضرب نقطي، يطبق softmax على الدرجات، يمزج قيمها |
| متعدد الرؤوس (Multi-head) | تقسيم d_model=128 إلى 4 رؤوس × 32 بُعدًا. كل رأس يتعلم علاقات مختلفة (بناء، موضع، دلالة) |
| القناع السببي (Causal mask) | الرمز i لا يستطيع الانتباه للرموز i+1, i+2... المستقبل مُقنَّع بـ -∞ قبل softmax → وزن 0 |
| التدفق المتبقي | الدخل + خرج الانتباه + خرج التغذية الأمامية. يخلق طريقًا سريعًا للتدرج يجعل الشبكة عميقة |
| الخسارة | الإنتروبيا المتقاطعة: مدى مفاجأة النموذج بالحرف التالي الصحيح. log(65) ≈ 4.17 = تخمين عشوائي |
| الحيرة (Perplexity) | e^الخسارة. "بين كم حرف كان النموذج يختار فعليًا؟" أقل = أفضل |
| استئصال الرأس (Head ablation) | تعطيل رأس واحد في كل مرة. قياس زيادة الخسارة. هذا الفارق = ما ساهم به ذلك الرأس |

```bash
cd phase1_transformer
python3.14 train.py        # ~5 دقائق على M1/M2 — يتدرب على شكسبير، يحفظ الأوزان
python3.14 tutorial.py     # تفاعلي، اضغط Enter عند كل قسم
```

## المرحلة 2 — مشفر الرؤية

### السؤال
كيف تعالج النماذج متعددة الوسائط مثل Claude وGPT-4V الصور؟

### الإجابة
اقطع الصورة إلى قطع ذات حجم ثابت. افلت كل قطعة في متجه. اسقط على نفس البُعد مثل رموز النص (d_model). أضف رموز الصورة قبل رموز النص. أدخل كل شيء في نفس المحوّل. رموز الصورة والنص تنتبه لبعضها بحرية — النموذج يرى الصورة والنص في آنٍ واحد من خلال آلية انتباه واحدة.

### ما تبنيه
- `PatchEmbedder` — يقطع صورة 64×64 إلى قطع 8×8 → 64 متجه قطعة من 256 بُعدًا
- `MiniVisionEncoder` — Vision Transformer (ViT): رموز القطع تنتبه لبعضها بلا قناع سببي
- `VisionLanguageProjector` — طبقتان خطيتان تعكسان vision_dim (256) → d_model (128)
- `MultiModalMiniGPT` — نموذج لغوي موسَّع لقبول صورة كرموز مُضافة في البداية

```bash
cd phase2_vision
python3.14 tutorial_vision.py     # لا حاجة للتدريب — يعمل على صور اصطناعية
```

## المرحلة 3 — الاستدلال (سلسلة التفكير)

### السؤال
كيف "يفكر" النموذج قبل الإجابة؟ ما هو آلية التفكير الموسَّع لدى Anthropic؟

### الإجابة
`<think>` مجرد رمز. تم تدريب النموذج على بيانات حيث ظهر `<think>` قبل خطوات الاستدلال و`</think>` قبل الإجابات الصحيحة. عند الاستدلال يُطابق النموذج النمط: "بعد `<think>`، توليد الاستدلال؛ بعد `</think>`، توليد الإجابة." رموز التفكير تتراكم في نافذة السياق. عندما يولد النموذج الإجابة النهائية، يعود لمسوَّدته الخاصة. هذا هو الآلية بالكامل — لا وحدة استدلال منفصلة.

### ما تبنيه
- مجموعة بيانات اصطناعية: نفس المسائل بتنسيقين — إجابة مباشرة مقابل سلسلة تفكير
- `model_direct` — مدرَّب بدون رموز تفكير، يقفز مباشرة للإجابة
- `model_cot` — مدرَّب مع `<think>...</think>`، يتعلم استخدام المسوَّدة
- مقارنة الدقة + تحليل الانتباه داخل كتلة التفكير

```bash
cd phase3_reasoning
python3.14 train_reasoning.py      # ~2 دقيقة — يدرّب نموذجَي direct و COT
python3.14 tutorial_reasoning.py   # قارن بينهما، شاهد الانتباه، اختبر الدقة
```

## المرحلة 4 — شخصيات العملاء

### السؤال
كيف يصبح نفس النموذج عملاء مختلفين بشخصيات وقدرات مختلفة؟

### الإجابة
الشخصية (Persona) هي رموز تُضاف في بداية كل دخل. النموذج لا يملك مفتاح وضع داخلي. يرى التسلسل الكامل للرموز — بادئة الشخصية + رسالة المستخدم — ويولد نصًا مشروطًا بكل ذلك. بادئة الشخصية تُنشّط أنماطًا مختلفة تعلمها من بيانات التدريب.

### الشخصيات الخمس في العمل (نفس السؤال: "هل يجب أن آخذ أسبرين لصداع؟")

```
المساعد العام        → "نعم، 325-650 ملغ مع الماء. تجنب إذا كنت على مضادات التخثر."
المستشار الطبي       → "النظر في التشخيصات التفريقية أولًا. استبعاد SAH للبداية المفاجئة..."
المعلم السقراطي      → "ما نوع الصداع؟ هل جربت الترطيب أولًا؟"
محامي الشيطان       → "التناول الانعكاسي للأسبرين يتجاهل السبب الجذري. هل أنت رطّب؟"
الشاعر              → "خذ القمر الأبيض الصغير / مع الماء / دعه يذيب العاصفة."
```

نفس النموذج. نفس السؤال. خمس إجابات. فرق بادئة واحدة فقط.

```bash
cd phase4_agents
python3.14 tutorial_personas.py
ANTHROPIC_API_KEY=sk-... python3.14 tutorial_personas.py   # ردود Claude المباشرة
```

## المرحلة 5 — متعدد العملاء + MCP

### السؤال
كيف يتواصل عدة عملاء، يستخدمون الأدوات، ويشكّلون شبكة؟

### الإجابة
Model Context Protocol (MCP) هو المعيار المفتوح لـ Anthropic لتوصيل العملاء بالأدوات. الأداة مجرد دالة Python مسجَّلة على خادم. العميل يستدعيها بتوليد نمط رمز منظَّم. الحلقة تنفذ الدالة، تضيف النتيجة للسياق، وتستدعي النموذج مجددًا. عدة عملاء يتشاركون ناقل رسائل يمكنهم تفويض المهام، جمع النتائج، وتوليف الإجابات — شبكة.

### طوبولوجيا الشبكة

```
المستخدم → [المنسق]
               │ send_message → [الباحث]   → search() → حقائق
               │ send_message → [المحلل]    → calculate() → أرقام
               │ get_messages ←───────────────────────────────────
               │ تلخيص
المستخدم ← [المنسق] الإجابة النهائية
```

```bash
cd phase5_multiagent
python3.14 tutorial_multiagent.py
ANTHROPIC_API_KEY=sk-... python3.14 tutorial_multiagent.py   # شبكة عملاء مباشرة
```

## توسيع الوسائط

نفس البنية تمتد لأي وسيط يمكن فيه ترميز البيانات:

| الوسيط | المرمِّز | ما يمثله الرمز | تسمية التدريب |
|---|---|---|---|
| النص | BPE / حرف | جزء كلمة / حرف | الرمز التالي |
| الصورة | قطعة + خطي | قطعة بكسل 8×8 | وصف الصورة |
| الصوت | EnCodec / DAC | ~12ms من الصوت | الرمز الصوتي التالي |
| الموسيقى | EnCodec + وسم النوع | مقطع موجة | المقطع التالي |
| الفيديو | قطعة لكل إطار | قطعة مكانية-زمنية | الإطار التالي |
| البروتين | مفردات الأحماض الأمينية | بقية واحدة | البنية المطوية |
| المالية | فئات الأسعار | لقطة OHLCV | فئة السعر التالية |

## البنية المحورية (مشتركة عبر جميع المراحل)

```
الدخل (أحرف نصية، قطع صور، رموز صوتية، ...)
        │
        ▼
  الترميز → معرّفات صحيحة
        │
        ▼
  بحث في جدول التضمين + الترميز الموضعي
  كل معرّف رمز → متجه بعده d_model
        │
        ▼
  ┌─────────────────────────────────────┐
  │   TransformerBlock  × N_طبقات      │  N طبقات = N خطوات استدلال
  │                                     │
  │   LayerNorm(x)                      │
  │       ↓                             │
  │   MultiHeadAttention                │  الرموز تنتبه لبعضها
  │   Q = x·Wq,  K = x·Wk,  V = x·Wv  │  H رؤوس × head_dim = d_model
  │   scores = QKᵀ / √head_dim         │  مصفوفة انتباه (T×T)
  │   weights = softmax(scores + mask)  │  قناع سببي للغة
  │   out = weights · V                 │
  │       ↓ + متبقي                    │
  │   LayerNorm(x)                      │
  │       ↓                             │
  │   FeedForward                       │  كل رمز يعالج باستقلالية
  │   x → Linear(d_model→4·d_model)    │
  │     → GELU                          │
  │     → Linear(4·d_model→d_model)    │
  │       ↓ + متبقي                    │
  └─────────────────────────────────────┘
        │
        ▼
  LayerNorm → Linear(d_model → vocab_size)
        │
        ▼
  Logits → softmax → احتمالية الرمز التالي
```

هذه البنية وراء GPT-2، GPT-3، GPT-4، Claude، Llama، Qwen، Gemini، Mistral. الأبعاد وعدد الطبقات والرؤوس تختلف. الهيكل لا يتغير.

## المرحلة 7 — بنية الإحساس

خريطة المشروع أعلاه تغطي أسس الهندسة. المرحلة 7 تطبّقها على سؤال أصعب: ماذا يلزم لمنح نظام ذكاء اصطناعي هوية فريدة، مستمرة، ومشكَّلة بالخبرة؟

ثلاثة نقاط ضعف هندسية في نماذج اللغة الكبيرة الحالية تُحفّز العمل:

| الثغرة | المشكلة |
|-----|---------|
| **الفردانية** | كل نسخة نموذج متطابقة بايت-بايت — "من هذا؟" لا إجابة ذات معنى |
| **الاستمرارية** | الهوية تُعاد ضبطها عند نهاية الجلسة |
| **المرونة** | الأوزان مجمَّدة عند الاستدلال — الخبرة لا تستطيع تغيير النظام |

الحل بنية ثلاثية المستويات: **المستوى 1** (خلايا عصبية مدرَّبة بالتسميات) · **المستوى 2** (PersonalityConnectome — متجه فريد من 12 سمة ذو بذرة فريدة، ملف دماغ JSON مستمر، تحديث هيبيان محدود) · **المستوى 3** (MetaConnectome + InquiryLayer — اكتشاف الثغرات المصنَّفة، دالة منفعة مرجَّحة بالشخصية للاستفسار).

## لماذا MLX

MLX هو إطار تعلم الآلة من Apple المُحسَّن لبنية الذاكرة الموحَّدة لشرائح M-series. وحدة المعالجة المركزية ووحدة معالجة الرسومات تتشاركان نفس الذاكرة الفيزيائية — لا نسخ للبيانات بينهما. النماذج التي تتسع في الذاكرة العشوائية (جميعها هنا) تعمل بكفاءة بدون GPU مستقل.

الكود يتوافق مباشرة مع PyTorch: استبدل `mlx.core` بـ `torch`، و`mlx.nn` بـ `torch.nn`، أضف مناداة `.to(device)`. المفاهيم متطابقة. اختير MLX هنا لأنه يعمل بشكل جيد على Mac الذي لديك بالفعل.
